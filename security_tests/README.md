# Security Tests — Pruebas Ofensivas Controladas

Este directorio agrupa escaneos, pruebas de superficie de ataque y resultados de validación ofensiva del laboratorio.

## Objetivo
- Identificar exposición por zona.
- Registrar hallazgos con severidad y estado.
- Entregar evidencia reproducible para análisis Blue/Red Team.

## Alcance inicial Sprint 1
- Enumeración de puertos/servicios por zona.
- Validación básica de exposición esperada/no esperada.
- Registro de observaciones para hardening posterior.

## Entregables esperados
- Reportes de escaneo.
- Evidencia técnica (salidas de comandos, logs, capturas).
- Referencias al `docs/PROYECTO/Registro_Hallazgos.md`.

## Buenas prácticas
- No ejecutar pruebas fuera del alcance definido.
- Etiquetar cada evidencia con fecha, zona y responsable.
- Priorizar seguridad de laboratorio y trazabilidad.
