# Listado de Herramientas — Sprint 0

Inventario base de herramientas definidas para Readiness del Equipo A.

## Infraestructura y contenedores

| Herramienta | Versión objetivo | Uso |
| --- | --- | --- |
| Docker Engine / Docker Desktop | Estable (2026) | Ejecución de contenedores por zona |
| Docker Compose (plugin v2) | Estable (2026) | Orquestación de servicios |
| Git | Estable | Control de versiones y trazabilidad |

## Administración y operación

| Herramienta | Uso |
| --- | --- |
| PowerShell / Bash | Ejecución de comandos operativos |
| VS Code | Edición técnica y documentación |
| Navegador (Firefox/Chrome/Edge) | Pruebas funcionales de servicios web |

## Diagnóstico y pruebas

| Herramienta | Uso |
| --- | --- |
| `dig` / `nslookup` | Validación DNS |
| `curl` / `wget` | Validación HTTP/Proxy |
| `ftp` / cliente FTP | Validación FTP anónimo y autenticado |
| `nc` (netcat) | Verificación rápida de puertos |
| `nmap` | Enumeración de puertos y superficie |
| Wireshark / tcpdump (opcional) | Inspección de tráfico en troubleshooting |

## Equipamiento de red físico

| Componente | Uso |
| --- | --- |
| Switch L2 (VLAN) | Segmentación 10/20/40 |
| Router L3 | Inter-VLAN, ACL y NAT base |
| Firewall | Políticas perimetrales y control de salida |

## Criterio de cumplimiento

Se considera cumplido este entregable cuando:

1. Las herramientas críticas están identificadas.
2. Cada herramienta tiene un propósito operativo claro.
3. Las guías del repositorio usan el mismo set de herramientas.
