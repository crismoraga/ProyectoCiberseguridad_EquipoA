## Regla de trazabilidad

Todo cambio técnico relevante debe dejar:

1. Evidencia (comando/log/captura).
2. Registro documental asociado (bitácora o informe de sprint).
3. Estado actualizado en artefacto correspondiente.

## Estado de cumplimiento (Readiness)

- **Sprint 0:** completo a nivel documental y de planificación técnica.
- **Sprint 1:** repositorio y configuraciones listas para ejecución en sala.
- **Pendiente para cierre formal:** validación runtime en laboratorio físico + firma QA/Líder.
